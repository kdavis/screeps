# Keir's Screeps Repository

This repository is holding my "artificial intelligence" and aims to be fully automated.

## Progression

Each "room" should be considered it's own entity. In the event of an "emergency" or an attack, there should be methods to request help from other rooms.
In the event of a room being overpowered by an invasion, the room should prioritise extracting resources to safer rooms.

### Stages of Room Progression

#### 1. Settlement

No defenses at this stage.

All creeps are designated to resource gathering and developing simple infrastructure to accelerate development and progression.

#### 2. Village

At this stage, there must exist at least 1 "Soldier" to prevent small scale attacks.

In addition, preparations to walls, ramparts and roads between these doorways to other rooms should be created.

Initially, towers should also be created to protect storage containers and the spawn.

#### 3. Fortress

Soldiers are able to answer to other room's summons for protection.

At this stage, scouts should be created to venture into nearby rooms to start new settlements or report back for attack.

#### 4. Citadel

Fully self sufficient in terms if being attacked. Should now participate in markets.

### Creep Roles

#### Roadworker

Should be designed to _only_ build roads between two places of interest. If available, to build a dual lane road to allow bi-directional passage without slowing down creeps

#### Harvester

Should gather resources or energy from a source and deposit it into a container for storage

#### Archer

Should perform a ranged attack and be used for protecting workers from enemies

#### Soldier

Should be close range attack with a higher amount of health
